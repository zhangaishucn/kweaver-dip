# smart-data-interpretation（数据解读：趋势/异常/贡献/建议）

> **维护说明**：本文为数据解读子技能**完整正文**（在此目录统一维护）。**总编排入口**：[`../SKILL.md`](../SKILL.md)。

本 skill 的定位是：**把“已取到的数据结果”解读成可行动的洞察**。它与 `smart-reporting` 的区别是：

- `smart-reporting`：按固定报告框架把上游证据写成“可交付报告”（偏写作与组装）。
- `smart-data-interpretation`：在不新增取数的前提下，对“结果表”做**趋势、异常、贡献维度与建议**（偏分析解读）。

> 强约束：本 skill **不做取数**、不调用 `text2sql` / `smart-ask-data` / `smart-search-tables` / `execute_code_sync` / `json2plot` 等任何外部工具；只消费输入。

## 能力边界（MUST）

- **仅基于输入**：不得新增任何表/字段/口径/数值；不得“合理推测”未出现在输入中的数字。
- **不取数**：不生成 SQL、不执行 SQL、不调用任何外部接口或工具。
- **可复核**：输出中的每个关键数字（同比、环比、占比、TopN、异常阈值等）必须能在输入结果中逐条定位或由输入内数据做**可验算**的简单计算得到（并写清计算口径）。
- **不越权**：若用户问题需要“原因归因”但必须新增取数才能证明（例如需要更多维度字段/对比期/漏斗拆分），必须在“局限与下一步”中明确指出需要新增哪些查询，而不是编造原因。

---

## 输入契约（MUST）

输入必须来自 `smart-ask-data` 的最终交付原文，至少包含：

- **kn_id**（业务取数 KN）
- **SQL（原样）**
- **结果数据（原样）**（表格或 JSON 行数组）
- **最小口径**（与 SQL 一致：时间范围、过滤条件、分组粒度、排序/TopN 等）

若缺失任一项：本 skill 必须终止并列出“需补齐的输入清单”，不得编造解读。

## 输出模板（固化，必须按此章节顺序）

> 说明：以下模板用于“解读结果”，不是“取数交付”。结果表若行数较多，可仅展示前 N 行，但必须声明“输入是否截断/是否为 TopN”。

### 1. 解读摘要（1–3 条）

- 结论必须可回指输入结果中的具体行/列或可验算计算。

### 2. 口径与证据来源（必须）

- **kn_id**：<来自输入>
- **SQL（原样）**：引用输入 SQL（可再次粘贴或指向输入块）
- **结果集结构**：字段列表与行数（以输入为准；如输入未给总行数，禁止捏造）
- **口径复述**：时间范围、过滤条件、分组粒度、排序规则（严格与输入一致）

### 3. 趋势（如结果包含时间序列；否则写“不适用”并说明原因）

- **趋势描述**：上升/下降/波动/拐点（必须引用具体时间点与数值）
- **阶段划分（可选）**：仅当输入能支持（例如按月/按周）

### 4. 异常与可能解释（受限）

- **异常点**：列出异常时间点/分组项与数值（必须可回指）
- **可能解释**：仅允许两类：
  - **证据内解释**：例如“异常发生在某分组项占比骤增”，需能用输入结果证明
  - **假设（需新增取数验证）**：明确标注“假设”，并在“下一步”列出验证所需新增查询

### 5. 关键贡献维度（必须尽量给出）

根据输入结果的分组维度，给出：

- **Top 贡献项**：TopN（按输入排序为准；不得重新排序后与输入冲突）
- **贡献度表达**：若输入包含占比字段可直接引用；若无占比字段，仅可给“绝对值对比”或在可验算前提下给出占比计算（写清分母取法）

### 6. 建议（必须与证据绑定）

输出 3–7 条建议，按优先级分组：

- **立刻可做（不新增取数）**：基于输入结果可直接采取的动作或核查点
- **需要补数验证**：每条写清“要新增哪条查询/哪些字段/哪个对比期”

### 7. 局限与下一步（必须）

- 明确指出当前输入无法支持的结论（例如缺少对比期、缺少关键维度、结果被 TopN 截断等）
- 给出最小增量的“下一步取数清单”（以可 SQL 表达为准；由 `smart-data-analysis` 路由回 `smart-ask-data` 或 `smart-attribution-analysis` 执行）

## 适用与不适用（路由提示）

- 适用：用户说“解读/分析这份取数结果”“帮我看趋势/异常/贡献/建议”，且**不要求新增取数**。
- 不适用：用户要“归因分析报告/系统性原因拆解/多维度下钻”，需要多子问题取数与证据包 → 应由 `smart-data-analysis` 路由到 `smart-attribution-analysis`（取数）并再交给 `smart-reporting`（报告）。
