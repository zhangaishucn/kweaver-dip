# smart-search-tables playbook（按需加载）

适用于复杂“找表/找视图”任务的防漏执行手册。主流程请先遵循 `smart-search-tables.md`，本文件在需要补检、扩词、精确核验时加载。

> **与主流程步号对齐**：`smart-search-tables` 固定为第 **5–11** 步；**最终四列表格与流程完成**在第 **11** 步（第 9 步需执行视图详情权限校验）。本 playbook 只补强第 7 步前后的检索策略，不改变主文件中的步序与门禁。

## 通用防漏检索模板（跨场景复用）

### A. 四段式检索流程

1. **主主题检索（粗召回）**
   - 结构：`<部门/区域> + <业务主题> + <核心动作>`
   - 示例动作词：`认定`、`许可`、`审批`、`备案`、`监管`（按场景替换）

2. **同义与别名扩展（补召回）**
   - 对核心动作词做同义扩展：`主词 -> 同义词/近义词/别称`
   - 示例：`认定 -> 核定/确认`，`变更 -> 调整/更正`

3. **子流程拆分检索（补细分）**
   - 将流程拆为子环节并分别检索：`申请`、`新办`、`延续`、`变更`、`注销`、`补办`、`年审`、`换证`（按场景取用）
   - 防止只命中“总称表”或只命中“单环节表”

4. **精确名反查与缺失词回查（交付前校验）**
   - 用户点名的表名必须逐条精确检索验证。
   - 结果发布前执行一次“缺失词回查”（见 C 节验收规则）。

### B. 关键词设计模板（可直接替换）

- **主题词组**：`<业务对象>`、`<事项类型>`、`<监管动作>`
- **同义词组**：`<主词A>/<同义A1>/<同义A2>`、`<主词B>/<同义B1>/<同义B2>`
- **子流程词组**：`<环节1>/<环节2>/<环节3>/...`
- **限定词组**（可选）：`<部门>`、`<区域>`、`<系统域>`、`<时间>`
- **排除词组**（可选）：`<不纳入口径关键词>`

推荐组合方式（分别检索后合并去重）：
- `限定词 + 主题词 + 主动作词`
- `限定词 + 主题词 + 同义动作词`
- `限定词 + 主题词 + 子流程词`
- `限定词 + 用户点名精确表名/短语`

### C. 交付前验收规则（必须满足）

1. **去重完成**：按 `视图技术名称`（或 UUID）去重。
2. **覆盖检查**：主主题、同义词、子流程三类检索均至少有一轮结果。
3. **点名校验**：用户点名对象全部完成“存在性核验”（命中或明确无命中原因）。
4. **缺失词回查**：对本次任务的高风险漏词执行补检（由场景自定义）。
5. **结果标注**：在最终回复中标注“已执行二次校验/补检”。

### D. 输出口径建议

- 将结果按“总称对象 / 子流程对象 / 补充对象”分组展示，避免泛称与细分表混淆。
- 对名称相近对象增加一句口径说明（例如“该表属于 X 环节子流程”）。
- 若存在“泛称未命中但细分命中”，应显式说明映射关系，避免误判为缺表。

### E. 最小执行示例（可直接替换）

1. **准备参数（替换占位词）**
   - `部门/区域`：`<ORG>`
   - `业务主题`：`<TOPIC>`
   - `核心动作`：`<ACTION>`
   - `同义词组`：`<ACTION_SYNONYM_1>`, `<ACTION_SYNONYM_2>`
   - `子流程词组`：`<STEP_1>`, `<STEP_2>`, `<STEP_3>`
   - `点名对象`（可选）：`<EXACT_NAME>`

2. **最小检索词清单**
   - `<ORG> <TOPIC> <ACTION>`
   - `<ORG> <TOPIC> <ACTION_SYNONYM_1>`
   - `<ORG> <TOPIC> <ACTION_SYNONYM_2>`
   - `<ORG> <TOPIC> <STEP_1>`
   - `<ORG> <TOPIC> <STEP_2>`
   - `<ORG> <TOPIC> <STEP_3>`
   - `<ORG> <EXACT_NAME>`（仅当用户点名时）

3. **PowerShell 执行骨架（单词条复用）**

```powershell
$query = "<ORG> <TOPIC> <ACTION_OR_STEP>"
$payload = @{
  ot_id = "metadata"
  condition = @{
    operation = "or"
    sub_conditions = @(
      @{ field = "embeddings_text"; operation = "match"; value = $query }
      @{ limit_value = 1000; field = "embeddings_text"; operation = "knn"; value = $query; limit_key = "k" }
    )
  }
  limit = 30
}
$jsonArg = $payload | ConvertTo-Json -Depth 8 -Compress
$null = $jsonArg | ConvertFrom-Json
$jsonEscaped = $jsonArg -replace '"','\"'
kweaver context-loader query-object-instance $jsonEscaped
```

4. **最小验收**
   - 已覆盖：主主题 + 同义词 + 子流程。
   - 已执行：用户点名对象精确核验（如有）。
   - 已完成：按技术名/UUID 去重后输出最终清单。
